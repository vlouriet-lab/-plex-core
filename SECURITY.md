# Политика безопасности

## Поддерживаемые версии

| Версия | Поддержка          |
|--------|--------------------|
| 0.1.x  | ✅                 |

## Сообщить об уязвимости

Если вы обнаружили уязвимость безопасности — **не создавайте публичный issue**.

Отправьте подробное описание на: **vlouriet@gmail.com**

Укажите в письме:
- Описание уязвимости и возможные последствия
- Шаги для воспроизведения
- Версию библиотеки

Ответ — в течение 72 часов. После подтверждения уязвимости будет назначен срок исправления и,
при желании, вы будете упомянуты в списке благодарностей.

## Область ответственности

Сообщения принимаются по следующим компонентам:

- Криптографические протоколы (Double Ratchet, X3DH, Argon2id KDF)
- SQLCipher-хранилище (ключи шифрования, миграции)
- P2P-транспорт (iroh / QUIC, NAT traversal)
- FFI-граница (UniFFI, утечки памяти / типов через FFI)
- Relay-репутация и identity verification

## Вне области ответственности

- Уязвимости в сторонних зависимостях — сообщайте напрямую их мейнтейнерам
- Denial-of-service сценарии без реального вектора атаки
